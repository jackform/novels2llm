🟢 本地执行

Stage 1, 2, 4, 5, 6 — 纯 Python 本地处理：YAML 解析、jieba 分词、HanLP NER、别名消歧、NetworkX 图构建

🟣 LLM 调用

Stage 3 — 5 个 extractor 分别调用 DeepSeek/Claude API，每块 6000 字文本 + NLP hints 作为 prompt

✂️ 分块策略

章边界优先 → 单章>8000字则句边界切 → 200字重叠保持上下文 → 11块/本 (5万字小说)

🔄 跨块串联

各块独立提取 → 别名 name_map 消歧 → 关系/对话人名映射到规范名 → 图去重合并

⚠️ 当前缺口 (待实现)

缺失 对话缺少精确位置

dialogue 只有 chapter 字段，没有 char_offset / line_index，无法精确对齐原文位置。NLP 阶段已有对话在原文中的 start/end 位置，但未写入最终输出。

缺失 事件跨块去重

每块独立生成事件都以 ch1_evt1 开头编号，3 块会产生 3 个 ch1_evt1。需要跨块去重 + 全局重排序。

缺失 对话 ↔ 事件 无桥接

dialogue 和 event 各自独立提取，两者唯一的共同字段是 chapter。无法回答"这段对话发生在哪个事件中"。

缺失 关系类型校验

张淑惠被同时标注 sibling 和 lover（应是 parent+lover），没有对不合理的关系组合（如母子被标为兄弟）做校验。

缺失 批量处理 & 缓存

Stage 3 缓存机制代码已有但未触发，重复运行会重新调用 API。34 本小说批量处理逻辑未测试。
